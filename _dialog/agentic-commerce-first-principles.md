## Frame

If autonomous agents become the default surface for commerce, incumbents' moats reprice around three questions: **who owns the routing**, **who owns the transaction rail**, and **who owns post-purchase trust**.

## Routing

The agent picks the merchant. Whoever the agent trusts to enumerate the option set becomes the new "aisle." Right now that layer looks like it will fragment across model providers rather than consolidate under a single retailer.

## Transaction rail

Agent-initiated transactions demand programmatic reconciliation — refunds, disputes, and post-hoc auth changes. That favors payment networks with agent-friendly APIs over legacy checkout flows.

## Post-purchase

Trust degrades quickly if the agent picks wrong. Merchants with strong return logistics and clean SLAs are cheaper to route to; agents will learn this and re-weight accordingly.

## Where I'd look for opportunity

- Middleware SaaS that gives merchants agent-ready product feeds
- Payment infrastructure with programmatic dispute APIs
- Verticals with heavy compliance requirements the agents can't fully abstract

*This is a working note, not a thesis — probably wrong in specific places, and that's the point of putting it here rather than in the research section.*
